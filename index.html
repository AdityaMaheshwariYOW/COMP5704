Project Title: Low-dose X-ray CT Image Reconstruction on GPUs

Name: Aditya Maheshwari

E-Mail: adityamaheshwari@cmail.carleton.ca

Project Outline: This project focuses on model based iterative methods for XCT image reconstruction on GPUs. The startup paper (cuMBIR) provides a solution that allows each thread in a GPU to handle the reconstruction of one X-ray (slice) of the 3D object. Because slices have an irregular geometric relationship, and GPUs slow down significantly with irregular patterns of memory access, the paper suggests using a graph colouring algorithm to separate X-rays that have a higher degree of collisions (i.e minimize the number of overlaps on the voxels - 3D pixels - of the image that they are recreating). After these collisions are minimized, the paper suggests another graph partitioning algorithm so that X-rays (threads) that access similar (but not the same) parts of the image are placed into the same warp to minimize non-coalesced memory accesses. Using this mapping, the GPU is then used to iteratively update the image by using either iterative coordinate descent or stochastic gradient descent to minimize an objective function which when minimized produces a high quality and smooth image. In this project I will first attempt to recreate the results of the paper (but on different GPU architectures; they use [NVIDIA Volta 100 GPU, resulting in a 1.48X speedup]), and then further explore the first two ideas by testing different parallelism granularities and setting different hyper-parameters for ICD and SGD. I will also test a new version of ICD for this problem because I think it can be further parallelized, specifically because of the sparse structure of XCT data. After that, if time permits I will explore to see if there is a way to parallelize this (or another) graph colouring/partitioning step potentially so that we can try running this problem in batches. Finally, (again if time permits) I would like to use the new method on some slightly different datasets just to see if they can actually generalize.
